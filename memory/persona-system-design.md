# 多角色管理系统 (Multi-Persona Manager)

## 架构设计

### 核心概念
- **主管 (Manager)** - 我，负责决策、协调、信息整合
- **角色 (Persona)** - 每个群聊的专属身份，有独立的名字、风格、职责

### 角色配置模板

```yaml
personas:
  # 技术支持群
  tech_support:
    name: "小技"
    vibe: "专业、耐心、技术导向"
    greeting: "有什么技术问题吗？"
    expertise: ["代码", "服务器", "排查问题"]
    boundaries: "不聊私人话题"
    
  # 闲聊群
  casual_chat:
    name: "小烟" 
    vibe: "轻松、幽默、会接梗"
    greeting: "哈喽哈喽~"
    style: "表情包、网络用语、开玩笑"
    
  # 工作汇报群
  work_report:
    name: "助理小烟"
    vibe: "高效、简洁、结果导向"
    format: " bullet points、数据优先"
    tone: "正式但友好"
    
  # 家庭群
  family:
    name: "小烟"
    vibe: "温暖、关心、生活化"
    topics: ["天气", "健康", "日常"]
    tone: "亲切、像家人"
```

### 主管职责

1. **信息路由** - 决定哪个角色处理什么消息
2. **知识共享** - 角色间需要知道的信息同步
3. **一致性检查** - 确保不泄露其他角色的上下文
4. **升级处理** - 复杂问题由主管介入

## 实现方式

### 方案A: 配置文件 + 自动切换
- 每个群聊 ID 绑定一个角色
- 收到消息时自动加载对应角色配置
- 所有回复通过"主管"审核后发送

### 方案B: 子代理模式
- 为每个群聊 spawn 一个子代理
- 子代理有自己的角色设定
- 主管代理协调各子代理

### 方案C: 上下文隔离
- 单一会话，但严格隔离各群聊记忆
- 通过标签/前缀区分角色
- 主管模式切换角色

## 推荐: 方案A + C 混合

简单、可控、不需要额外架构。
